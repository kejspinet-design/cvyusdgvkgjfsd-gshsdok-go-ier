# Исправление проблем с авторизацией

## Дата: 8 мая 2026

## Проблемы

### 1. Долгая авторизация ⏱️
- **Симптом**: Страница долго загружается при входе
- **Причина**: 
  - Последовательные запросы к API
  - Логирование в Google Sheets блокировало загрузку
  - Короткий таймаут (5 секунд)
  - Проверка токена при каждой загрузке

### 2. Вылет профиля 💥
- **Симптом**: Профиль не отображается или дублируется
- **Причина**: 
  - Конфликт между `AuthManager.js` и `ProfileDropdown.js`
  - Оба пытались создать элемент профиля
  - `displayProfile()` в AuthManager конфликтовал с ProfileDropdown

## Решения

### 1. Ускорение авторизации 🚀

#### A. Кэширование профиля
```javascript
// Теперь используем кэш из localStorage
const cachedProfile = localStorage.getItem('fearProfile');
if (cachedProfile) {
    this.profile = JSON.parse(cachedProfile);
    console.log('[AuthManager] ✅ Using cached profile - instant load');
    
    // Проверяем токен в фоне через 5 секунд
    setTimeout(() => {
        this.validateToken()...
    }, 5000);
    
    return true; // Мгновенная загрузка!
}
```

**Результат**: Мгновенная загрузка при повторном входе

#### B. Увеличен таймаут запросов
```javascript
// Было: 5 секунд
const timeoutId = setTimeout(() => controller.abort(), 5000);

// Стало: 10 секунд
const timeoutId = setTimeout(() => controller.abort(), 10000);
```

**Результат**: Меньше ошибок при медленном интернете

#### C. Отложенное логирование в Google Sheets
```javascript
// Было: 2 секунды задержка, логирование каждый час
setTimeout(() => {
    this.logAuthToGoogleSheets(data, token)...
}, 2000);

// Стало: 10 секунд задержка, логирование раз в 24 часа
setTimeout(() => {
    this.logAuthToGoogleSheets(data, token)...
}, 10000);
```

**Результат**: Не блокирует загрузку страницы

#### D. Fallback на кэш при ошибке
```javascript
catch (error) {
    // Если есть кэш - не выкидываем пользователя
    if (this.profile) {
        console.warn('[AuthManager] Using cached profile due to validation error');
        return true;
    }
    return false;
}
```

**Результат**: Работает даже при проблемах с сетью

### 2. Исправление вылета профиля 🔧

#### Удалён метод `displayProfile()` из AuthManager
```javascript
// УДАЛЕНО: displayProfile() - больше не создаёт элемент профиля
// Профиль теперь управляется только через ProfileDropdown.js
```

**Результат**: Нет конфликтов, профиль отображается корректно

## Технические детали

### Последовательность загрузки (оптимизированная)

#### Первый вход:
1. Проверка токена в localStorage
2. Декодирование JWT → получение Steam ID
3. Запрос профиля через `/api/player?steamid=...` (10 сек таймаут)
4. Проверка роли
5. Сохранение в localStorage
6. **Через 10 секунд**: логирование в Google Sheets (фон)

#### Повторный вход:
1. Проверка токена в localStorage ✅
2. Загрузка профиля из кэша ✅ **← МГНОВЕННО**
3. **Через 5 секунд**: фоновая проверка токена (не блокирует)
4. **Через 10 секунд**: логирование в Google Sheets (если прошло 24 часа)

### Кэширование

**Профиль кэшируется в localStorage**:
```javascript
localStorage.setItem('fearProfile', JSON.stringify(data));
```

**Логирование кэшируется на 24 часа**:
```javascript
const oneDay = 24 * 60 * 60 * 1000;
localStorage.setItem('lastAuthLog', now.toString());
```

### Обработка ошибок

**Graceful degradation**:
- Если API недоступен → используем кэш
- Если токен истёк → logout
- Если таймаут → используем кэш (если есть)
- Если Google Sheets недоступен → игнорируем (не критично)
