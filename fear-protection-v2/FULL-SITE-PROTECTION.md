## Как работает защита:

### 1. Автоматическая проверка при загрузке страницы

`RoleAccessControl.js` автоматически выполняется при загрузке каждой страницы:

```javascript
// Автоматическая проверка доступа
if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', () => {
        if (!window.location.pathname.includes('login.html') && 
            !window.location.pathname.includes('admin-access-test.html')) {
            roleAccessControl.checkAccess();
        }
    });
}
```

### 2. Проверка токена

```javascript
async checkAccess() {
    const token = localStorage.getItem('fearToken');
    
    if (!token) {
        // Нет токена → редирект на login.html
        this.redirectToLogin();
        return false;
    }
    
    // Декодируем токен и проверяем роль
    const payload = JSON.parse(atob(token.split('.')[1]));
    const steamId = payload.client_id;
    
    // Загружаем профиль и проверяем роль
    const profile = await fetch(`/api/player?steamid=${steamId}`);
    
    if (!this.hasAccess(profile)) {
        // Роль не разрешена → показываем "Доступ запрещён"
        this.showAccessDenied(profile);
        return false;
    }
    
    return true;
}
```

### 3. Что происходит без токена:

1. Пользователь открывает любую страницу сайта
2. `RoleAccessControl.js` проверяет наличие токена в `localStorage`
3. **Если токена нет** → автоматический редирект на `login.html`
4. **Если токен есть, но роль не разрешена** → показывается страница "Доступ запрещён"
5. **Если токен есть и роль разрешена** → страница загружается нормально
